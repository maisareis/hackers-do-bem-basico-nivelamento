# Aula 2 – Principais Frameworks

## 📌 O que é um Framework
- Referência normativa com técnicas, ferramentas e conceitos
- Base para estabelecer programa de segurança da informação
- Oferece processo para implementar salvaguardas

## 📌 Importância dos Frameworks
- Criam modelos de construção de programas de segurança
- Estabelecem criação de políticas
- Gerenciam riscos e mantêm proteção contínua
- Avaliam portas de entrada, transferência e saída de dados

## 📌 ISO 27001
- Padrão internacional para gestão de segurança da informação
- Estabelece SGSI (Sistema de Gestão de Segurança da Informação)
- Lançada em 2005, abordagem de melhoria contínua
- **Objetivo:** proteger pilares CIA
- **Conteúdo:** SGSI + Anexo A (93 controles)
- **Anexo A:** políticas, organização, RH, gestão de ativos, controle de acesso, criptografia, segurança física, operações, comunicações, etc.

## 📌 CIS Controls
- Center for Internet Security
- Diretrizes para proteger aplicações e dados
- Controles priorizam menor número de ações com melhor custo-benefício
- **Grupos de Implementação (IG):**
  - **IG1:** 56 salvaguardas básicas para todas empresas
  - **IG2:** +74 salvaguardas (com equipe de TI)
  - **IG3:** +23 salvaguardas (dados sensíveis, conformidade)

## 📌 NIST CSF
- National Institute of Standards and Technology
- Padrões, diretrizes e práticas para gerenciamento de riscos
- Flexível para qualquer organização ou setor
- **5 funções:** Identificar, Proteger, Detectar, Responder, Recuperar
- **23 categorias** → **108 subcategorias** com controles
- **Implementação:** definir escopo, orientar, criar perfil atual, avaliar riscos, criar perfil alvo, analisar lacunas, executar plano

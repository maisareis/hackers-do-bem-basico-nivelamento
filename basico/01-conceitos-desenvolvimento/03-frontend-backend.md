# Aula 3 – Conceitos de front-end e back-end

## 📌 Front-end
- Parte visual e interativa do site/aplicativo
- O que o usuário vê e usa
- **UX Design** → usabilidade simples e intuitiva
- **Protótipos** → rascunhos da interface

## 📌 Back-end
- Código que conecta internet com banco de dados
- Gerencia conexões e alimenta aplicações
- **Principais linguagens:** PHP, Java, C#, JavaScript, Python, Ruby

## 📌 Funcionalidades do Back-end
- Gestão de bancos de dados
- Desenvolvimento/integração de APIs
- Segurança da informação
- Trabalho conjunto com UX/UI
- Testes, criação de bibliotecas
- Planejamento e manutenção da estrutura do site
- Acesso ao banco de dados para guardar/resgatar informações
- Fornecimento de dados para o front-end

## 📌 API
- Application Programming Interface
- Conjunto de padrões que permite comunicação entre sistemas diferentes
- Permite compartilhar dados, funções e serviços

## 📌 Requisições HTTP
- **GET** → solicita dados
- **POST** → envia dados para processamento
- **PATCH** → modifica partes de um recurso
- **PUT** → atualiza um recurso
- **DELETE** → remove um recurso

## 📌 Análise de riscos
- Identificar ameaças, vulnerabilidades, impactos e probabilidades
- Definir estratégias de mitigação, prevenção e resposta

## 📌 Testes de segurança
- Verificar se aplicação atende aos padrões de segurança
- Devem ser realizados em todas as fases do desenvolvimento

## 📌 Autenticação e autorização
- **Autenticação** → verifica identidade do usuário (senha, token, biometria)
- **Autorização** → define nível de permissão
- Garantem que apenas usuários legítimos acessem recursos

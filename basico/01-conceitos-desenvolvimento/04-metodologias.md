# Aula 4 – Metodologias de desenvolvimento

## 📌 Importância das metodologias
- Organização, comunicação, qualidade garantida
- Menos surpresas, adaptação a mudanças
- Trabalho em equipe, cumprimento de prazos
- Clientes felizes, melhoria contínua

## 📌 Metodologia Cascata
- Desenvolvimento sequencial em fases fixas
- Uma fase só começa quando a anterior termina
- **Indicado para:** projetos bem definidos desde o início
- **Fases:** Requisitos → Design → Implementação → Testes → Manutenção

## 📌 Desenvolvimento Ágil
- Abordagem iterativa e colaborativa
- Entrega contínua de software funcional
- Adaptação a mudanças, satisfação do cliente
- **Indicado para:** projetos que exigem flexibilidade
- **Princípios:** indivíduos e interações, software funcionando, responder a mudanças

## 📌 Scrum
- Divide projeto em iterações chamadas sprints
- **Ciclo de vida:** Planejamento → Daily Scrum → Revisão → Retrospectiva

## 📌 XP (Extreme Programming)
- Ênfase em qualidade de código e colaboração
- **Ciclo:** Planejamento → Pequenos lançamentos → Feedback → Retrospectiva

## 📌 Desenvolvimento Incremental
- Divide projeto em partes independentes e funcionais
- Entrega funcionalidades adicionais em cada incremento

## 📌 Desenvolvimento Lean
- Busca eficiência eliminando desperdícios
- Prioriza entrega de valor ao cliente
- **Características:** identificação de necessidades, resposta rápida a mudanças, feedback contínuo

## 📌 TDD (Test-Driven Development)
- Testes automatizados antes do código de produção
- **Passos:** Escrever teste → Falhar → Implementar → Executar testes → Refatorar
